2021.08.21
338 words 1 min

#概述

nextword

#使用

  1. 第一步 下载 nextword-data，也就是英文常用单词库,下载压缩包，解压到本地
  2. 第二部 下载nextword 程序 根据系统选择assets，本地是mac64位的选择nextword_0.0.5_drawin_amd64.tar.gz

#设置

You need to install nextword-data and set "NEXTWORD_DATA_PATH" environment variable. It available at https://github.com/high-moctane/nextword-data

  1. 本人用的zsh
    所以在~目录下加入export NEXTWORD_DATA_PATH=/Users/xxx/Documents/nextword/nextword-data-small

  2. 还需要在usr/bin/sh加入nextword命令 否则会提示/user/bin/sh:nextword: Cmomand not found
    由于我们下载的nextword已经是一个编译好的程序，所以只需要cp /User/xxx/Documents/nextword/nextword /usr/local/bin

adding-a-new-executable-to-the-path-environment-variable

到这一步，本地的nextword在命令行应该可以跑起来了

然后再vscode 按照 nextword插件就能使用我们刚刚下载的nextword-data单词库了

不过有点坑的是vscode-nextword会覆盖原有的autocompletion 所以在js等文件中并不好使

替换插件vscode-dic-completion

该插件在新版的vscode支持js语法了

然后再vscode的config.json可以指定使用的单词库"dictCompletion.externalUserDictFiles": ["/Users/xxx/Document/google-1000-english-master/20k.txt"],

单词库下载地址google-10000-english
该插件和大部分语法提示兼容
